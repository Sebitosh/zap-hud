Экран опций HUD

Включить при использовании рабочего стола ZAP

Если установлено, HUD будет внедряться в HTML-ответы при использовании ZAP Desktop. По умолчанию это значение true.

Включить при использовании ZAP в режиме демона

Если установлено, HUD будет внедряться в HTML-ответы, когда ZAP запускается в режиме демона или в «безголовом» режиме. По умолчанию это значение равно false, так как многие люди используют ZAP для автоматизации, внедрение HUD не поможет и может вызвать существующие тесты сломать.

Show the HUD welcome screen when a browser is opened

When set the HUD Welcome screen, which includes a link to the tutorial, will be shown whenever a browser that is proxying through ZAP is opened. The welcome screen includes an option to not show the screen again, if that is selected then this is the only way to make the welcome screen appear again.

Enable the HUD only for URLs that are in scope

If selected then only URLs flagged as 'in scope' in ZAP will have the HUD injected into them. The plan is to expose this option via the HUD, but until that time if the option is enabled then you will need to add URLs to the scope via the ZAP Desktop.

Когда эта опция выбрана, значок HUD на панели инструментов рабочего стола будет иметь небольшое наложение «цель» на нижняя правая сторона.

Включить сообщения внутри домена

Разрешить отправку сообщений из целевого домена в домен ZAP. Это необходимо для таких функций, как отображение количество скрытых полей и отображение оповещения, связанного с конкретным полем. Если вы считаете, что целевой домен может быть вредоносным, вы можете отключить эту опцию. HUD не доверяет никаким сообщениям из целевого домена, но вредоносный сайт потенциально может делать слегка раздражающие вещи, такие как вызывая неожиданное отображение предупреждений ZAP. Вам нужно будет перезапустить браузер после изменения этого параметра, чтобы он вступил в силу.

Удалить CSP с целевых страниц

В настоящее время HUD не будет работать, если на целевых веб-сайтах используется сильная политика CSP. По умолчанию ZAP отключает CSP, отключение этой опции может привести к поломке HUD.

Режим разработки

Если установлено, устанавливаются различные параметры, которые могут быть полезны при отладке HUD, в том числе:
  • Установка уровня отладки на стороне клиента на DEBUG
  • Отправка журнала на стороне клиента на вкладку ZAP Output
  • Добавлен инструмент «Ошибки HUD», который показывает любые зарегистрированные ошибки.

Базовый каталог
